Background

What happened & why this works

Terrashroom built their app on top of Blynk, an IoT platform that handles the connection between the app and the device. When the company shut down, their Blynk server went offline — but the device itself is perfectly fine. It still boots, still runs its grow automation, and still tries to connect to a server. We just need to give it one to talk to.

The key insight: The Blynk local server is open source, free to run on any PC or Raspberry Pi, and the Terrashroom device will connect to it without any modifications to the hardware or firmware.

Overview

How it works

Three parts that work together:

1

DNS Spoofing

Tell your network to redirect console.terrashroom.io to your own PC or phone instead of the real (now-dead) server. The device never knows the difference.

2

Local Blynk Server

Run an open-source Blynk server locally. It accepts the device's connection and logs the auth token — a 32-character string that identifies your specific unit.

3

Controller App

A single HTML file that runs in any browser. Enter your auth token and local server IP — full control restored. Fans, misting, dome, lights, heater, setpoints, everything.
